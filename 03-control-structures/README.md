# Pattern Matching <a id="pattern-matching"></a>

__Pattern matching__ in Java is a technique used to execute code, if and only if a certain criteria is met. To
understand how and why pattern matching has been introduced, let's consider the following example:

```java
public class Application {
    
    public static void main(String[] args) {
        Application.patternMatching(12);
    }

    public static void patternMatching (Number randomNumber) {
        
        if (randomNumber instanceof Integer) {
            final Integer parsedNumber = (Integer)randomNumber;

            System.out.println(randomNumber + " is an instance of Integer class!");
            return;
        }

        System.out.println(randomNumber + " is an instance of another class!");

    }

}
```

Analyzing code examples like this, Java engineers identify a common pattern, that is:

```java
if (randomNumber instanceof Integer) {
    final Integer parsedNumber = (Integer)randomNumber;
}
```

Therefore, an useful mechanism to implements, would consists in reducing the number of lines required to check if a
variable is an instance of a specific class, and then perform the correct casting. The result of these considerations is
the following code:

```java
if (randomNumber instanceof Integer parsedNumber) {
    System.out.println(randomNumber + " is an instance of Integer class!");
    return;
}
```

The interest part of this code is that: `randomNumber instanceof Integer parsedNumber` is evaluated as a boolean
expression. Therefore, it is possibile to concatenate this expression with other boolean expression, in order to enhance
the check condition of the pattern matching:

```java
public class Application {
    
    public static void main(String[] args) {
        Application.patternMatching(12);
    }

    public static void enhachedPatternMatching (Number randomNumber) {
        
        if (randomNumber instanceof Integer parsedNumber && parsedNumber > 10) {
            System.out.println(randomNumber + " is an instance of Integer class, and is greater than 10!");
            return;
        }

        if (randomNumber instanceof Integer parsedNumber && parsedNumber <= 10) {
            System.out.println(randomNumber + " is an instance of Integer class, and is smaller or equal than 10!");
            return;

        }

        System.out.println(randomNumber + " is an instance of another class!");

    }

}
```

Some considerations are mandatory for this construct: "What happens with variables having value `null`?", "Does pattern
matching works for every type?". 

1. Variables having value `null` are always evaluated as `false` in pattern matching comparison.
2. Only compatible types can be used for pattern matching, i.e.: you cannot check that a variable declared as `Number`
   is an instance of `String`, while it is possibile to check that the same variable is an instance of `Object`.

Based on the consideration made on point 2, the following code wont compile! `Number cannot be converted to String` will
be the output message generated by the compiler:

```java
public class Application {
    
    public static void main(String[] args) {
        Application.patternMatching(12);
    }

    public static void invalidPatternMatching(Number randomNumber) {
        
        if (randomNumber instanceof Integer parsedNumber && parsedNumber > 10) {
            System.out.println(randomNumber + " is an instance of Integer class, and is greater than 10!");
            return;
        }

        if (randomNumber instanceof Integer parsedNumber && parsedNumber <= 10) {
            System.out.println(randomNumber + " is an instance of Integer class, and is smaller or equal than 10!");
            return;
        }

        if (randomNumber instanceof String parsedNumber) {
            System.out.println(randomNumber + " is an instance of String!");
            return;
        }

        System.out.println(randomNumber + " is an instance of another class!");

    }

}
```

Last but not least, __pattern variables__ (i.e. variables parsed inside the pattern matching operator) possesses __flow
scope__. It means that the variable will be visible only inside the block of the `if` statement, neither inside the
`else` block.
